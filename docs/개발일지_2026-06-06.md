## 4. 오늘 배운 점 / 메모

- GitHub Pages 소스가 "GitHub Actions" 모드면 워크플로우 파일이 없을 때 **조용히 404**가 된다 — `has_pages` API와 Actions 실행 이력으로 진단 가능
- GitHub Pages는 배포 후에도 브라우저 캐시 때문에 구버전이 보일 수 있다 → 정적 자원에 버전 쿼리를 붙이는 것이 효과적
- 카카오톡은 OG 정보를 자체 캐시한다 → 변경 시 [카카오 공유 디버거](https://developers.kakao.com/tool/debugger/sharing)에서 캐시 초기화
- CSS `animation-fill-mode: both`는 종료 후에도 transform을 점유해 JS 인라인 스타일과 충돌 → `backwards`로 해결
- git에서 파일 삭제는 히스토리에 남는다 — 민감 문서는 처음부터 올리지 않는 것이 최선
